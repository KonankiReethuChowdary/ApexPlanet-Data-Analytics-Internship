# Task 1: Data Dictionary

| Column Name | Data Type (Initial) | Description | Business Relevance |
| :--- | :--- | :--- | :--- |
| `order id` | String / Object | A unique alphanumeric identifier assigned to each individual sales transaction. | Critical for tracking total order volume, preventing double-counting, and mapping items to transactions. |
| `order date` | String / Object | The raw date and timestamp text when the customer placed the order. | Used to analyze seasonal sales trends, peak purchasing hours, and day-over-day growth. |
| `customer id` | String / Object | A unique identifier code linked to a specific customer's account. | Essential for customer lifetime value (LTV) analysis, cohort tracking, and identifying repeat buyers. |
| `customer name`| String / Object | The full name of the customer who completed the purchase. | Used for personalizing marketing outreach or validating user account details. |
| `age` | Integer / Float | The age of the customer at the time of purchase. | Helps identify target demographic segments and age-based spending habits. |
| `gender` | String / Object | The customer's gender identity (e.g., Male, Female, etc.). | Crucial for gender-based product preferences and targeted advertising campaigns. |
| `city` | String / Object | The geographical location/city where the customer resides or where the order is shipped. | Used for regional sales distribution analysis and logistics planning. |
| `product` | String / Object | The specific name or model of the item purchased. | Helps track which inventory items are top sellers versus slow-moving stock. |
| `category` | String / Object | The broader department or group the product belongs to (e.g., Electronics, Apparel). | Allows high-level tracking of product department performance and profit margins. |
| `quantity` | Integer | The number of units of the product purchased in the transaction. | Used to calculate total units sold and manage inventory supply needs. |
| `unit price` | Float | The individual cost of a single unit of the product. | Helps monitor price sensitivity, discounts, and revenue generation metrics per item. |
| `total sales` | Float | The final monetary amount generated by the order (usually quantity x unit price). | The core target metric used to track company revenue, growth, and profitability. |
